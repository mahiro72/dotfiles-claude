# PR ドラフト作成

これまでの会話で行った作業内容からPRタイトルと概要を作成します。

**重要**: このスキルが呼ばれた場合、確認は不要です。直ちに出力してください。

## 出力形式

コードブロック（```markdown）で囲んで出力すること。

```markdown
## PRタイトル
<type>: <簡潔な説明>

## 概要
<1-3行で何をしたか>

## 背景（必要な場合のみ）
<なぜこの変更が必要だったか>
```

## タイトルのtype

- `feat`: 新機能
- `fix`: バグ修正
- `refactor`: リファクタリング
- `docs`: ドキュメント
- `chore`: 設定・雑務

## 方針

- シンプルに書く
- 変更ファイル一覧やテスト確認事項は不要
- 背景は複雑な変更の場合のみ記載
