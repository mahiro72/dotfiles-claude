# PR ドラフト作成

これまでの会話で行った作業内容からPRタイトルと概要を作成します。

**重要**: このスキルが呼ばれた場合、確認は不要です。直ちに出力してください。

## 出力形式

PRタイトルはコードブロックの外に出力し、概要はコードブロック（```markdown）で囲む。

```
## PRタイトル
<type>: <簡潔な説明>

## 概要
```markdown
<1-3行で何をしたか>
```
```

## タイトルのtype

- `feat`: 新機能
- `fix`: バグ修正
- `refactor`: リファクタリング
- `docs`: ドキュメント
- `chore`: 設定・雑務

## 方針

- シンプルに書く
- 変更ファイル一覧やテスト確認事項は不要
- 背景はユーザーが「背景も書いて」と指示した場合のみ追加
